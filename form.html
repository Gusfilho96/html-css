Formulários:

  • <form> Forma mais basica de criar um formulário.
  • action Informa para onde será enviado o formulário.
  • get O metodo get serve para enviar informações não sigilosas, até 3000 bites.
  • post Apesar de ser um pouco mais seguro, ele ainda permite que as informações sejam acessadas.
  • input: Podemos criar vários tipos de input, alguns deles são:
    • color Esse tipo de input premite que o usuario selecione alguma cor, podemos usar value para iniciar com alguma cor.
    • range Esse metodo funciona muito bem para presquisas de satisfação, podemos colocar valor min ou max e o value para o nivel inicial.
    • file Esse metodo permite enviar algum arquivo, ao utilizar esse recurso é importante utilizar o metodo Post, caso contrario o arquivo não será enviado.
    • checkbox O input de tipo checkbox serve para uma marcação simples, um exemplo seria a marcação de esportes que a pessoa pratica. Utilizar label nesse caso server tanto para o navegador saber a quem aquele input está ligado, e servira também para que o usuario utilize a palavra para marcar a checkbox.
    • radio Pode ser utilizado quando deve-se marcar apenas uma opção, um exemplo seria a seleção do sexo. Nesse caso o name deve ser o mesmo em ambos dentro daquele fieldset, já label pode ser utilizado normalmente. Outra coisa importante é o value, um simples value="M" ou "F" ajudara na hora do submit.
    • select Esse elemento tem vários atibutos interessantes, podemos colocar option com as opções e o value que serão enviados ao serem selecionados. Podemos também adicionar optgroup para adicionar grupos dentre as escolhas, o atributo label pode então ser adicionado para informar o nome do grupo. Outro elemento é o selected utilizado dentro de option para que aquela opção já apareça selecionada. Obs: select é um elemento separado, não um input:select
    • text O input desse tipo é bastante util, ele pode servir para enviar um texto simples ou criar uma lista que se adapta ao usuário. Para criar uma lista usando text, adicione o atributo list e crie um elemento datalist, adicione um id que sera lincado o atributo list e dentro dela adicione optioncom todos os elementos da lista que você quer.
    • textarea Serve para que o usuário tenha uma area onde ele pode escrever uma mensagem, acredito que isso é mais utilizado para sugestões no caso de serviços prestados. Para criar essa área, basta adicionar o elemento textarea, nela você pode definir o numero de colunas e linhas mostradas, esse tamanho pode ser configurável pelo usuario
  • label Serve para fazer a ligação com o input, essa ligação não tem mudança visual.
  • <fieldset> Funciona para agrupar os campos contidos nessa tag, isso pode ser usado em conjunto com a tag <legend> dentro, possibilitando que todos os dados pessoais, por exemplo fiquem dentro de um campo.
  • Alguns atributos:
    • required
    • min/max lenght
    • placeholder
    • autocomplete
    • min/max
